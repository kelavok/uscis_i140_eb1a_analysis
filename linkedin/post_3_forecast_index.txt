How was the parent-category pending correction transferred to E11 and NIW?

The annual snapshots only provide long-term maturation data for EB1 and EB2. The target programs, however, are E11 and NIW. To connect them, I used quarterly RADP data from FY2022-FY2026 Q1.

For every quarter I calculated:

program denial odds = program denials / program approvals
parent denial odds = parent denials / parent approvals
program-parent multiplier = program odds / parent odds

Across the available quarters:

• E11 denial odds were approximately 2.31x EB1 odds
• NIW denial odds were approximately 2.60x EB2 odds

The program and parent series moved closely together. The correlation of log denial odds was 0.980 for E11/EB1 and 0.962 for NIW/EB2.

The final model combines two time-varying components in log-odds space:

program pending denial odds
= parent pending-resolution odds trend
x program-parent multiplier trend

For FY2026 Q1, the model estimates the following outcome mix specifically for pending inventory:

• E11 pending-to-denial share: 61.6% (44.5%-76.2%)
• NIW pending-to-denial share: 73.8% (52.7%-87.7%)

After combining those pending estimates with outcomes already decided, the projected aggregate final denial shares are:

• E11: 60.2%, scenario range 45.8%-72.5%
• NIW: 72.8%, scenario range 53.0%-85.8%

These are model-based scenario ranges, not individual case probabilities. RADP pending is an inventory stock rather than a clean filing cohort. USCIS aggregate files also do not provide RFE history, premium-processing status, case strength, or adjudicator-level controls.

The correct use is therefore comparative: monitor the index after every new quarter, update the ranges, and backtest them as later snapshots mature.
